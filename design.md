# テトリス対戦 設計書

## 1. 概要

ブラウザで動作するオンライン対戦テトリスゲーム。  
単一のHTMLファイル（`index.html`）で構成され、Cloudflare Pages上にデプロイして公開する。  
Firebase Authenticationによるユーザー認証、Cloud Firestoreによるデータ永続化、PeerJS（WebRTC）によるP2P対戦通信を備える。

### 公開URL

`https://tetris-game-ce8.pages.dev/`

### 技術スタック

| 項目 | 技術 |
|------|------|
| フロントエンド | HTML / CSS / JavaScript（単一ファイル） |
| 描画 | Canvas API |
| 認証 | Firebase Authentication（メール＋パスワード） |
| データベース | Cloud Firestore |
| P2P通信 | PeerJS（WebRTC） |
| ホスティング | Cloudflare Pages |
| リポジトリ | GitHub（`bco-hiraguchi/tetris-game`） |

## 3. ゲームロジック

### 3.1 ボード仕様

| パラメータ | 値 |
|-----------|-----|
| 列数（COLS） | 10 |
| 行数（ROWS） | 20 |
| ブロックサイズ（BLOCK） | 30px |
| キャンバスサイズ | 300 x 600 px |

### 3.2 テトロミノ

7種類の標準テトロミノを使用。各ピースに固有のカラーを割り当て。

| 種類 | 色 | カラーコード |
|------|----|-------------|
| I | シアン | `#00f0f0` |
| J | 青 | `#0000f0` |
| L | オレンジ | `#f0a000` |
| O | 黄 | `#f0f000` |
| S | 緑 | `#00f000` |
| T | 紫 | `#a000f0` |
| Z | 赤 | `#f00000` |
| お邪魔 | グレー | `#888888` |

### 3.3 操作方法

| キー | 動作 |
|------|------|
| ← | 左移動 |
| → | 右移動 |
| ↑ | 右回転（壁キック対応） |
| ↓ | ソフトドロップ（+1点/マス） |
| Space | ハードドロップ（+2点/マス） |
| C | ホールド |

### 3.4 スコアリング

#### ライン消去

| 同時消去数 | 基本点 |
|-----------|--------|
| 1ライン | 100 |
| 2ライン | 300 |
| 3ライン | 500 |
| 4ライン（テトリス） | 800 |

実際のスコア = 基本点 × 現在のレベル

#### ドロップボーナス

- ソフトドロップ: 1点/マス
- ハードドロップ: 2点/マス

### 3.5 レベルとスピード

- レベル = `floor(消去ライン数 / 10) + 1`
- 落下間隔（ms） = `max(50, 1000 - (レベル - 1) × 80)`

| レベル | 落下間隔 |
|--------|---------|
| 1 | 1000ms |
| 2 | 920ms |
| 5 | 680ms |
| 10 | 280ms |
| 12+ | 50ms（最速） |

### 3.6 ホールド機能

- Cキーで現在のピースをホールド領域に保存
- ホールドにピースがある場合、現在のピースと交換
- 1回のドロップにつき1度のみ使用可能
- 使用済み時はホールド表示が半透明（alpha=0.3）になる
- ピース固定後（自然落下・ハードドロップ）にリセット

### 3.7 ゴーストピース

現在のピースの落下先をボード上に半透明（alpha=0.2）で表示。

### 3.8 壁キック

回転時に壁や他ブロックと衝突する場合、左右に1マスずらして回転を試みる。

### 3.9 ゲームオーバー条件

- 新しいピースの出現位置が既存ブロックと重なった場合
- ピース固定時にボード上端（y < 0）にブロックがある場合
- お邪魔ブロック追加で現在のピースが衝突した場合

### 3.10 ゲームオーバー処理

`handleGameOver()` 関数に統一。以下の2箇所から呼び出される：

1. **ゲームループ内**: 自然落下によるゲームオーバー検出時
2. **キー入力ハンドラ内**: ハードドロップ等の操作による即座のゲームオーバー検出時

これにより、どのタイミングでゲームオーバーになっても確実に結果画面が表示される。

## 4. 対戦機能

### 4.1 通信方式

PeerJS（WebRTCラッパー）を使用したP2P通信。サーバーレスで動作。

### 4.2 接続フロー

1. **ホスト**: 6桁のルームコードを生成し、PeerJSサーバーにPeer IDとして登録
   - Peer ID形式: `tetris-battle-{6桁コード}`
   - コード文字セット: `ABCDEFGHJKLMNPQRSTUVWXYZ23456789`（紛らわしい文字を除外）
2. **ゲスト**: ルームコードを入力し、ホストのPeer IDに接続
3. 接続確立後、双方がユーザー名を`name`メッセージで送信
4. ホストが500ms後に`start`メッセージを送信し、両者のゲームを開始

### 4.3 P2Pメッセージ一覧

| type | 方向 | ペイロード | 説明 |
|------|------|-----------|------|
| `name` | 双方向 | `{ name }` | ユーザー名の通知 |
| `start` | ホスト→ゲスト | なし | ゲーム開始指示 |
| `state` | 双方向 | `{ board, score, level, lines, activePiece }` | ゲーム状態の同期（100ms間隔） |
| `garbage` | 双方向 | `{ lines }` | お邪魔ブロック送信 |
| `gameover` | 敗者→勝者 | なし | ゲームオーバー通知 |
| `rematch-request` | 双方向 | なし | 再戦リクエスト |
| `rematch-accept` | 双方向 | なし | 再戦承認 |

### 4.4 お邪魔ブロック

| 同時消去数 | 相手に送るお邪魔行数 |
|-----------|-------------------|
| 1ライン | 0（送信なし） |
| 2ライン | 1行 |
| 3ライン | 2行 |
| 4ライン | 3行 |

- お邪魔ブロックはボード最下段から追加
- 各行にランダムな位置に穴を1つ設ける
- お邪魔追加で現在のピースが衝突した場合、即ゲームオーバー

### 4.5 再戦ロジック

- `battleStarted`フラグで`startBattle()`の重複実行を防止
- 両者が再戦ボタンを押すと自動でゲーム開始
- 片方が先に押した場合、待機メッセージを表示
- 相手の切断時は自動勝利

### 4.6 状態同期

- 100ms間隔で自分のゲーム状態（ボード、スコア、アクティブピース）を相手に送信
- 相手のボードはリアルタイムで描画更新

### 4.7 ユーザー名表示

- ゲーム画面のボード上部に自分・相手のユーザー名を表示
- 対戦結果画面にも両者のユーザー名とスコアを表示
- ユーザー名はP2P接続確立時に`name`メッセージで交換

## 5. Firebase連携

### 5.1 Firebase Authentication

- 認証方式: メール + パスワード
- ユーザー属性: `displayName`にユーザー名を保存
- セッション管理: `onAuthStateChanged`でログイン状態を監視
- 新規登録時は`skipAuthRedirect`フラグで`onAuthStateChanged`の自動リダイレクトを抑制し、`updateProfile`完了後に手動遷移

### 5.2 Firestore データ構造

#### `users`コレクション

ドキュメントID: Firebase Auth UID

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `name` | string | ユーザー名 |
| `wins` | number | 対戦勝利数 |
| `losses` | number | 対戦敗北数 |
| `highScore` | number | ハイスコア |
| `totalGames` | number | 総プレイ数（ソロ＋対戦） |
| `createdAt` | timestamp | アカウント作成日時 |

#### `scores`コレクション

ドキュメントID: 自動生成

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `uid` | string | ユーザーUID |
| `name` | string | ユーザー名 |
| `score` | number | スコア |
| `level` | number | 到達レベル |
| `lines` | number | 消去ライン数 |
| `mode` | string | `'solo'` または `'battle'` |
| `date` | timestamp | プレイ日時 |

#### `matches`コレクション

ドキュメントID: 自動生成

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `uid` | string | ユーザーUID |
| `name` | string | ユーザー名 |
| `myScore` | number | 自分のスコア |
| `oppScore` | number | 相手のスコア |
| `result` | string | `'win'` または `'lose'` |
| `date` | timestamp | 対戦日時 |

### 5.3 データ保存タイミング

| イベント | 保存先 | 更新内容 |
|---------|--------|---------|
| ソロゲームオーバー | `scores` + `users` | スコア記録追加、totalGames+1、ハイスコア更新 |
| 対戦終了 | `matches` + `users` | 対戦記録追加、totalGames+1、wins/losses+1、ハイスコア更新 |

エラー時はコンソールにログ出力（`console.error`）。ランキング画面ではエラーメッセージをUIに表示。

### 5.4 プロフィール編集

- ユーザー名変更: Firebase Auth `updateProfile` + Firestore `users`ドキュメント更新
- メールアドレス変更: 再認証（`reauthenticateWithCredential`で現在のパスワード入力）→ `verifyBeforeUpdateEmail`で確認メール送信
- 成功/エラーメッセージをフォーム内に表示

### 5.5 ランキング

- Firestoreの`scores`コレクションを`score`降順でクエリ
- 上位20件を表示
- ユーザー名、スコア、モード（ソロ/対戦）を一覧表示
- エラー時はエラーメッセージを表示

- カウントダウン中はキー入力を無効化（`countingDown`フラグ）

## 8. デプロイ

### 8.1 ホスティング

- Cloudflare Pages
- GitHubリポジトリ（`bco-hiraguchi/tetris-game`）と連携
- `main`ブランチへのpushで自動デプロイ
- ビルドコマンド: なし（静的ファイル）
- 出力ディレクトリ: `/`
- ローカル開発ブランチ: `tetris-deploy`（pushは `git push origin tetris-deploy:main`）

### 8.2 Firebase設定

- プロジェクトID: `test-project-21811`
- Authentication: メール/パスワード有効
- Firestore: `asia-northeast1`リージョン（推奨）
- Firestoreセキュリティルール: 認証済みユーザーのみ読み書き可能に設定推奨

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```
